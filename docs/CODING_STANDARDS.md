# TradeX Coding Standards

## General
- Every file must compile with 0 errors and 0 warnings.
- Every module has a single responsibility.
- Every function should have a clear purpose.
- Use meaningful names for variables and functions.

## File Headers
Each source file should include:
- Module name
- Build number
- Short description

## Git
Commit every completed module.

Commit format:
build001.x: short description

## Documentation
Major changes must be recorded in CHANGELOG.md.

## Testing
Every module must compile successfully before being committed.
